## 0. 重新定位

### v1（之前）vs v2（現在）

| 維度 | v1 商業產品版 | v2 個人學習版 |
|---|---|---|
| 使用者數 | 多人，需要註冊 | 一人（你自己） |
| 認證 | NextAuth + 登入頁 | 無 |
| 資料庫 | Supabase PostgreSQL | SQLite（本地檔案） |
| 設定步驟 | 5 個（Supabase、Anthropic、NextAuth、Binance、GitHub）| 1 個（Anthropic） |
| 上線目標 | Vercel 公開網站 | 本地 localhost |
| 主要功能 | 辯論室 | 辯論室 + **學習路徑 + AI 教練 + 練習題 + 里程碑** |
| 啟動時間 | 30 分鐘 | **8 分鐘** |

### 核心理念

> **這個產品的存在目的不是讓你「更會用 AI」，是讓你「成為能穩定獲利的交易者」。**

AI 不是工具，是**教練 + 對手 + 裁判**三合一。

## 1. 8 個 Level 完整設計

### Level 0：認識市場與自己

**時長**：3-5 天  
**目標**：在學任何技術之前，先想清楚你為什麼來、能承受多少痛。

**AI 教練會問你的 5 個問題**：
1. 你為什麼想學交易？「想賺錢」不算答案——具體一點。
2. 如果你投入 1 萬塊，全虧光了，你會不會崩潰？
3. 你能不能接受「學會交易後，前 2 年仍然不賺錢」？
4. 你目前一週能投入多少小時在這件事上？
5. 你身邊有沒有人會反對你做這件事？怎麼處理？

**互動練習**：
- 寫一份「我的交易動機書」（300-500 字）
- AI 會用蘇格拉底式問題挑戰你——如果你寫「我想財務自由」，AI 會反問「對你來說，多少錢才叫財務自由？什麼時間之內達到？」

**通過標準**：
- 完成動機書並通過 AI 的 3 輪反問仍不動搖
- 在系統內填好「我能承受的最大虧損金額」與「我能承受的最大連虧次數」

**過了這級你會知道**：
你不是來「賭錢」的。如果動機是「想快速翻身」，AI 會明確告訴你**這個系統不適合你**，建議你去做別的事。

### Level 1：看懂 K 線、量、趨勢

**時長**：1-2 週  
**目標**：你看到任何一張 K 線圖，能在 10 秒內說出「這是上升、下降、還是盤整」。

**AI 教練的 12 堂課**：
1. K 棒是什麼？開高低收四個價格的視覺化
2. 紅 K 綠 K 為什麼這樣分（台灣 vs 國際相反）
3. K 棒形態：長紅、長黑、十字星、錘子、吊人線
4. 為什麼下影線比上影線重要
5. 成交量代表什麼？量價背離
6. 趨勢的 3 種狀態：上升、下降、盤整
7. 高點低點：什麼叫高點、什麼叫關鍵高點
8. 趨勢線：怎麼畫，畫錯怎麼辦
9. 突破與假突破：怎麼分辨
10. 多時間框架的概念：日 K 看到的趨勢、週 K 看到的可能相反
11. 為什麼短期看價、長期看量
12. 你的第一個「觀察清單」：每天看 3 檔，記錄你的觀察

**互動練習（核心）**：
- AI 從歷史 K 線資料庫隨機挑 10 張（已知後續走勢），考你「這張你判斷上升還是下降？」
- 你回答後，AI 顯示「實際後續走勢」，告訴你對在哪、錯在哪
- 累積到 30 張、正確率 70% 以上才能進下一級

**通過標準**：
- 30 張 K 線練習正確率 ≥ 70%
- 連續 7 天觀察 3 檔商品並寫下判斷

**過了這級你會知道**：
你能看圖了。但**看圖不等於會交易**——下一級才是關鍵。

---

### Level 2：風控與停損（這級是分水嶺）

**時長**：1-2 週  
**目標**：你能在 30 秒內算出任何交易的合理倉位、停損價、目標價。

**AI 教練會用大量數據打你的臉**：
- 為什麼 80% 的散戶虧錢？不是因為看錯方向，是**沒有停損**
- 賺錢需要「賺多賠少」，不是「賺多次」
- R:R（風險報酬比）的數學：勝率 50%、R:R 1:2 = 長期賺；勝率 70%、R:R 1:1 = 長期不賺
- **單筆風險絕對不能超過總資金的 2%**——為什麼這個數字是 2%？

**12 個必須會的公式**：
1. 單筆最大可虧 = 總資金 × 風險比例（建議 1-2%）
2. 倉位大小 = 單筆最大可虧 ÷ |進場價 − 停損價|
3. R:R = |目標價 − 進場價| ÷ |進場價 − 停損價|
4. 期望值 = 勝率 × 平均賺 − (1 − 勝率) × 平均賠
5. 連虧停止：3 連虧 → 強制停手 24 小時，5 連虧 → 停手 1 週
6. ...（後面 7 個進階公式）

**互動練習（最殘酷的一級）**：
- AI 給你 20 個情境：「總資金 100,000、進場 81,000、停損 79,800、目標 84,000，請算倉位」
- 必須**5 秒內**算對。算錯一次 AI 不會罵你；算錯 5 次以上，AI 強制你回到「為什麼要算這個」的根本問題
- 同時練習：對 5 個情境給出**「不該交易」的判斷**——例如 R:R 不到 1:1.5 時應該拒絕

**通過標準**：
- 20 道計算題正確率 100%
- 5 個「不該交易」情境全部判斷正確
- 寫一份「我的風控規則」文件並承諾遵守

**過了這級你會知道**：
**「不交易」也是一種交易**。一個月真正出手 3-5 次的人，往往賺得比每天出手的人多。

### Level 3：第一次 AI 辯論實戰

**時長**：2-3 週  
**目標**：你能完整跑通「觀察 → 辯論 → 決策 → 筆記 → 復盤」的循環，並從中找出自己的盲點。

**這級是 v1 版本就有的核心功能**——但現在它在學習路徑的中段，前面的 L0-L2 是準備工作。

**12 個主題**：
1. 開啟你的第一場辯論：BTC 為例
2. 怎麼讀 Bull 的論點？標的是「強/中/弱」證據而非結論
3. 怎麼讀 Bear 的論點？特別注意 rebuttal 部分
4. 怎麼讀 Judge 的分數？分數低於 5 就是「沒看法」
5. R:R 不到 1:2 = 不該交易
6. 怎麼寫交易筆記：3 個必填欄位
7. 1h 復盤：你還沒進場前，市場變了多少？你後悔嗎？
8. 4h 復盤：如果你進場了，會怎樣？
9. 1d 復盤：你的判斷對了還是錯了？對在哪？錯在哪？
10. 連續 7 天紀錄你的「情緒指數」與「決策一致性」
11. 找出你最常犯的 1 個錯誤類別（情緒/技術/風控/資訊）
12. 第一個月總復盤

**互動內容**：
- 真實開辯論（這就是 v1 版本的功能）
- 寫筆記
- 完成 1h/4h/1d 三輪復盤
- AI 自動分析你的錯誤模式

**通過標準**：
- 完成 20 場辯論
- 每場辯論都對應一份筆記
- 全部 60 次復盤（20 場 × 3 時點）完成
- 寫一份「我最常犯的錯誤」反思文件

**過了這級你會知道**：
你的弱點不是「看不懂市場」，是「**看懂後管不住手**」或「**只看到自己想看的證據**」。下一級教你怎麼把直覺變成可驗證的規則。

### Level 4：建立你的選股策略

**時長**：2-3 週  
**目標**：把你過去 L3 學到的「我會被什麼樣的標的吸引」變成 3 個明確的規則集。

**AI 教練的 8 堂課**：
1. 直覺 vs 規則：為什麼專業交易員都有 SOP
2. 策略 = 條件樹：AND / OR 巢狀邏輯
3. 從「我覺得好」到「PE < 15 AND 殖利率 > 4」
4. 排序與限制：每天最多看 5 檔，怎麼選？
5. 策略掃描的盲點：你的條件太寬還是太窄？
6. 月度策略復盤：勝率多少才算好策略？
7. 策略迭代：什麼時候該調整、什麼時候該砍掉
8. 多策略組合：3 個互補策略 > 1 個完美策略

**互動內容**：
- 你建立 3 個個人策略（基於 L3 紀錄分析你的偏好）
- 每天 09:00 系統執行掃描
- 結果列表你只能對 5 檔開辯論（培養篩選紀律）
- 月底 AI 分析每個策略的表現

**通過標準**：
- 建立 3 個策略並運行 30 天
- 至少 1 個策略的勝率高於「亂選」基準
- 寫一份「我的選股哲學」文件

**過了這級你會知道**：
你不需要每天看 100 檔——你需要的是**今天值得花時間的那 5 檔**。

---

### Level 5：跨市場進階

**時長**：3-4 週  
**目標**：你能在台股、美股、加密三個市場之間切換，並理解各自的特性。

**8 堂課**：
1. 台股特性：漲跌停、T+2、月營收、三大法人、融資融券
2. 美股特性：盤前盤後、earnings、Fed FOMC、13F
3. 加密特性：24/7、資金費率、ETF 流入、鏈上資料
4. 跨市場資金流：BTC 漲時美股科技股的關係
5. 各市場最適合的時間框架
6. 各市場最常見的陷阱
7. 怎麼選你「主修」的市場
8. 一週只專注一個市場：避免分心的紀律

**互動內容**：
- 三市場各完成 10 場辯論
- 比較同一週期內三市場的 Judge 結論
- 月底寫「我最適合哪個市場」反思

**通過標準**：
- 三市場各 10 場辯論完成
- 對三市場特性能口頭/書面解釋給朋友聽
- 選定你的「主市場」並寫出選擇理由

**過了這級你會知道**：
**沒有「最好的市場」，只有「最適合你的市場」**。年輕人通常適合加密（24/7、波動大、學習快），但需要對應心理素質。

---

### Level 6：多時間框架對齊

**時長**：3-4 週  
**目標**：對同一檔商品，你能說出撥頭皮、波段、中線、長線四個視角的不同結論，並判斷它們是否互相衝突。

**8 堂課**：
1. 為什麼時間框架不同，結論不同（同一個 BTC $81,126，波段觀望、中線偏多）
2. 撥頭皮的勝率為什麼極低（80% 散戶輸錢）
3. 波段：散戶最該專注的時間框架
4. 中線：基本面 + 技術面的甜蜜點
5. 長線：耐心與情緒的真正試煉
6. 多時間框架對齊：日線多頭 + 週線多頭 = 高勝率
7. 框架衝突：日線偏多但週線偏空時怎麼辦
8. 你的「主時間框架」：選一個專注

**互動內容**：
- 對同一檔商品依序開 4 種時間框架辯論
- AI 強制你回答：「這四個結論為什麼不同？」
- 練習辨識「框架對齊」的高勝率情境

**通過標準**：
- 對 10 檔商品完成 4 框架辯論（共 40 場）
- 找出 3 個「四框架完全對齊」的情境並交易
- 寫「我選定的主時間框架」與理由

**過了這級你會知道**：
**多數虧損來自「框架混淆」**——「波段進場、中線出場、長線套牢」。

---

### Level 7：職業化框架

**時長**：持續累積（建議至少 6 個月）  
**目標**：你有一份完整的「交易系統文件」，能傳給朋友看，且你自己連續 3 個月遵守它。

**這級沒有課程**，只有 5 件事：

1. **撰寫你的交易系統文件**（5000 字以上）
   - 我的市場與時間框架
   - 我的選股規則
   - 我的進場規則（精確到「滿足哪些條件才能下單」）
   - 我的出場規則（停損 + 停利兩種）
   - 我的加減碼規則
   - 我的心態規則（連虧停手、賺到一定程度該休息嗎）
   - 我的復盤頻率與檢視標準

2. **持續每日紀律**（不能斷）
   - 每天 09:00 看策略掃描結果
   - 對符合條件的標的開辯論
   - 寫筆記、復盤

3. **每月績效復盤**
   - 我的勝率、平均賺賠比、最大回撤
   - 我違反系統幾次？為什麼？
   - 我的系統需要調整嗎？

4. **每季交易框架更新**
   - 過去 3 個月學到什麼新東西
   - 哪條規則該砍、哪條該加

5. **找一個交流對象**
   - 不是「跟單群組」，是能挑戰你思路的人
   - 每月至少 1 次「把你的交易給對方看」

**通過標準（最嚴格）**：
- 交易系統文件完整且可讀
- 連續 3 個月勝率 > 50% **且**遵守系統 > 90%
- 寫一份「我這 6-12 個月的學習旅程」回顧

**過了這級你會知道**：
你不再是學員。你是有自己系統的交易者。
**但「持續遵守系統」比「達到這級」更難**——這是一輩子的功課。

## 2. 簡化版技術架構

### 砍掉的東西
```
- NextAuth（不需要登入）
- @auth/prisma-adapter
- bcryptjs
- /login, /signup 頁面
- middleware.ts 的路由保護
- User, Account, Session, VerificationToken 4 個 model
- Supabase（改用 SQLite）
- DATABASE_URL, DIRECT_URL, NEXTAUTH_URL, NEXTAUTH_SECRET 4 個環境變數
```

### 新增的東西
```
+ /learn 路由（學習儀表板）
+ /learn/[level] 路由（每個 Level 的內容）
+ /practice/[type] 路由（互動練習：kline / risk / debate / etc.）
+ AI Tutor system prompt（蘇格拉底式教練）
+ 互動練習資料（30+ 張歷史 K 線，已標註答案）
+ Milestone 系統（每個 Level 的通過判定）
+ LearningProgress model（你的學習軌跡）
+ LearningJournal model（你的反思紀錄）
```

### 改變的東西
```
~ Prisma datasource 從 postgres 改為 sqlite
~ 全部 entity 拿掉 userId（單一使用者）
~ Dashboard 顯示「當前 Level + 進度 + 下一步」
~ 啟動只需要 1 個環境變數：ANTHROPIC_API_KEY
```

### 啟動流程

```bash
unzip crown-market-room-v2.zip
cd crown-market-room
npm install
# 只要一行：
echo 'ANTHROPIC_API_KEY=sk-ant-api03-...' > .env.local
npx prisma db push        # SQLite 自動建檔案
npm run seed:all          # 灌入加密貨幣 + 歷史 K 線練習題
npm run dev
# 開 http://localhost:3000
```

**3 分鐘設定 + 5 分鐘申請 Anthropic key = 8 分鐘完工**。

## 3. 簡化後的資料庫 Schema

只剩 10 個 model（v1 有 15+ 個）：

```prisma
// === 商品與市場資料 ===
model Product            // 加密 / 美股 / 台股商品
model PriceSnapshot      // 凍結價快照
model OHLCVDaily         // 日 K 歷史
model ProductMetrics     // 指標快照

// === 辯論與筆記 ===
model DebateSession      // AI 辯論 session（拿掉 userId）
model TradingNote        // 交易筆記（拿掉 userId）
model ReviewLog          // 復盤紀錄（拿掉 userId）

// === 個人學習（新增） ===
model LearningProgress {
  id              String   @id @default("singleton")
  currentLevel    Int      @default(0)
  completedLevels Int[]
  milestonesPassed Json    @default("[]")
  updatedAt       DateTime @updatedAt
}

model LearningJournal {
  id        String   @id @default(cuid())
  level     Int
  type      String   // reflection | milestone | lesson_note
  content   String
  createdAt DateTime @default(now())
}

model PracticeAttempt {
  id          String   @id @default(cuid())
  level       Int
  practiceType String  // kline_judge | risk_calc | etc.
  question    Json
  myAnswer    Json
  correct     Boolean
  createdAt   DateTime @default(now())
}

// === 選股策略 ===
model ScreenerStrategy   // 拿掉 userId
model ScreenerResult     // 同 v1
```

## 4. 新增的 AI 角色：Tutor

除了 Bull / Bear / Judge，新增第 4 個角色：

**AI Tutor**（教練）
- 模型：Claude Sonnet 4.6
- 任務：用蘇格拉底式提問引導你，而不是直接給答案
- 觸發場景：
  - 每個 Level 開始時的「動機檢核」
  - 你練習答錯時的「為什麼錯了」分析
  - 你寫反思時的「再追問一次」
  - 月度復盤時的「你最大的盲點是什麼」

Tutor 的核心 prompt 規範：
- **不准直接給答案**——必須用提問引導
- 必須挑戰使用者，**不能取悅**
- 如果使用者顯示不健康訊號（追漲殺跌、賭博心態），**直接點出**

這個角色比 Bull/Bear/Judge 更重要，**因為它讓「教學」這件事真的發生**。
